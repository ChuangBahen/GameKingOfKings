## ADR-002: 錯誤處理模式

### 狀態
已接受

### 背景
需要統一的錯誤處理方式，讓程式碼更一致且易於維護。

### 決策
使用 Result Pattern 來處理可預期的錯誤，Exception 只用於真正的例外情況。

### 選項考量

#### 選項 A: 傳統 Exception
- 優點：C# 原生支援，使用簡單
- 缺點：效能開銷，容易被忽略

#### 選項 B: Result Pattern
- 優點：明確的錯誤處理，編譯期檢查
- 缺點：需要更多樣板程式碼

### 理由
Result Pattern 讓錯誤處理更明確，減少遺漏錯誤處理的可能性。

### 影響
- 所有 Service 方法回傳 Result<T>
- 需要統一的 Result 類別定義
